## 專案目標與範圍摘要
本專案目標是建立一套可監控學習作弊行為、支援主管審核決策、並保留完整稽核軌跡的合規系統。

系統核心能力包含以下三塊：
- 風險收件匣（Risk Inbox）：集中呈現被規則引擎標記的異常學習事件。
- 鑑識時間軸（Forensic Timeline）：重建單筆學習紀錄的操作行為與違規依據。
- 處理決策列（Resolution Action Bar）：提供主管做出核准、作廢重修、升級 HR 等結案處理。

本文件以 Web 應用程式方式規劃，技術方向固定為前端 `HTML + CSS + JavaScript`、後端 `Python FastAPI`、資料庫 `PostgreSQL`，並明確列出資料模型、API、後端規則、前端頁面、測試與 demo 交付任務。
